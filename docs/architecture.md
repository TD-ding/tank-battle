# 架构设计

## 技术栈

- **HTML5 Canvas** — 游戏渲染，使用离屏缓存（offscreen canvas）优化地图绘制
- **原生 JavaScript** — 无框架依赖，单文件实现
- **Web Audio API** — 合成音效（振荡器 + 增益节点），无需音频文件
- **CSS3** — 界面覆盖层样式
- **localStorage** — 历史最高分持久化

## 文件结构

```
index.html    全部游戏代码（HTML + CSS + JS 单文件）
```

## 代码组织

单文件内按逻辑段落组织，各段以注释分隔：

| 段落 | 说明 |
|------|------|
| 主题配置 (THEME) | 所有颜色和视觉参数的集中配置 |
| 常量 | 瓦片大小、地图尺寸、速度、时间等游戏参数 |
| Canvas 初始化 | 画布、DOM 引用、错误提示 |
| 音效系统 | Web Audio API 合成音效，含静音和节流 |
| 地图 | 地图模板、初始化、基地保护墙生成 |
| 游戏状态 | 全局状态变量（分数、生命、计时器等） |
| 最高分 | localStorage 读写函数 |
| 工具函数 | 网格对齐、碰撞检测、移动合法性检查 |
| 道具 (Powerup) | 道具类、生成、拾取、效果应用 |
| 坦克 (Tank) | 坦克类，含移动、射击、绘制，以及 AI 参数 |
| 爆炸效果 (Explosion) | 时间驱动的爆炸动画，含碎片粒子 |
| 敌人生成 | 多出生点轮询、场上数量限制 |
| 碰撞检测 | 子弹与墙壁/坦克/子弹/基地的碰撞处理 |
| 敌人 AI | 方向切换、撞墙记忆、追踪概率、射击 |
| 地图渲染 | 离屏缓存绘制、基地护盾特效、水面动画 |
| HUD | Canvas 绘制的分数/生命/道具状态面板 |
| Overlay | 开始/暂停/胜利/失败界面 |
| 输入处理 | 键盘事件、方向键/Esc 阻止默认行为 |

## 核心设计决策

### 帧率无关

所有移动和计时基于 `dt`（毫秒），速度单位为 像素/秒。`dt` 上限为 50ms，防止切标签页回来后跳帧。暂停恢复时重置 `lastTime = 0`。

### 网格对齐

坦克只在转向时对齐到网格中心，直行时顺畅移动。这样既保证转向能进入通道，又不会让直行产生顿挫感。

### 离屏缓存

地图（砖墙、钢墙、基地）仅在 `mapDirty` 标记为 true 时重新绘制到离屏 canvas，每帧直接 `drawImage` 复制。水面动画因需要逐帧更新，单独在主 canvas 上绘制，且只遍历水面瓦片索引。

### 音效节流

同类音效设有最小间隔（射击 80ms、命中 50ms），防止密集交火时产生音频堆叠。静音通过 `muted` 全局标志控制。

### Tank ID

每辆坦克分配自增数字 ID（`nextTankId++`），子弹通过 `tankId` 引用发射者的数字 ID，而非对象引用。避免敌人被销毁后子弹仍持有悬空引用。
